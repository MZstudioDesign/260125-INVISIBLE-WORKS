## SECTION 3: Problem

### 목표
- 사용자의 상황을 **대신 설명**
- 감정 끌어올리기 ❌
- 불안 조성 ❌
- 해결책 암시 ❌
- **관찰자 시점**, "아는 얘기"로 고개만 끄덕이게

### 메인 카피
> "웹사이트는 비용이 크고, 과정이 복잡하며, 관리하기 어렵습니다."

### 연출: 3D 룰렛 스크롤
**좌측**에 둥근 느낌의 3D 원통형 텍스트 릴:
- perspective 효과로 원통처럼 보이게
- 현재 선택된 항목에 → 화살표 표시
- 자동으로 주르륵 스크롤

**텍스트 목록**:
```
도메인 / 호스팅 / SSL인증 / 보안 / 속도 / 구조 / 레이아웃 /
디자인 / 콘텐츠 / 반응형 / 접근성 / SEO / 퍼포먼스 /
브라우저호환 / 모바일최적화 / 유지보수 / 업데이트 / 백업 / 배포 / 관리
```

**전달 인상**: "신경 쓸 게 끝이 안 보인다"

## SECTION 4: Change

### 목표
- 웹사이트가 비싸고, 어렵고, 오래 걸리던 이유를 **정확히 짚음**
- "잘해서"가 아니라 **"거품을 빼서"** 가능해졌다는 인식

### 말해도 되는 표현
- 공정을 줄였다
- 거품을 뺐다
- 불필요한 단계를 없앴다
- 일을 단순하게 만들었다

### 말하면 안 되는 표현 ❌
- 싸다, 빠르다, 자동화, AI, 혁신

### 연출: 타임라인 블러
세로 타임라인에 5개 항목이 복잡한 과정처럼 보이다가, 스크롤에 따라 **빠르게** 블러/지워짐:

1. 전체가 복잡하게 느껴진 진행
2. 왜 필요한지 모르겠던 단계들
3. 결정할 게 너무 많았던 과정
4. 이해하기 어려웠던 질문들
5. 계속 늘어만 났던 수정 사항

### 중앙 카피
> "우리는 일을 줄였습니다.
> 대신 고민을 늘렸습니다."

### 남긴 것 (리스트)
- 꼭 필요한 판단만
- 설명이 필요한 지점만
- 결과로 이어지는 과정만

### 마무리 문장
> "그래서, 웹사이트를 만드는 방식이 달라졌습니다."

### 섹션 전환
**GradientHorizon**: 해가 뜨듯 **진한 블루** 그라데이션 지평선 (Taurus 예시처럼 강렬하게)

## SECTION 5: How We Do

### 목표
- 고객이 **어디까지 하면 되는지** 명확히
- 일이 어떻게 흘러가는지 **예측 가능**하게
- "쉽다" 말하지 않음, 대신 복잡해질 여지가 없는 구조를 보여줌

### 타이틀
> "이렇게 진행됩니다"

### 연출: 아코디언 스텝 (스크롤 시 자동 펼침)

#### 01 내용 전달
> "웹사이트에 들어갈 자료를 전달합니다."

#### 02 디자인 선택
> "디자인 시안 3종 중 하나를 선택합니다."

**내부에 세로로 긴 이미지 3장** (라이트박스 확대 가능)

**강조 카피 (무료 강조)**:
> "그리고, **여기까지 무료입니다.**"
> "디자인은, 보고 판단하셔도 늦지 않습니다."

- "무료"에 **강한 강조** (그라데이션, 색상 구분, 애니메이션 효과)

#### 03 웹사이트 완성
> "선택한 디자인으로 웹사이트를 정교하게 완성합니다."

## SECTION 6: Design Preview (Section 5 내부)

### 역할
- 시안 3개가 어떤 '결과물'인지 연결 설명
- 포트폴리오 없이도 "아, 이렇게 나오는구나" 즉시 이해
- 추상적인 '시안' 개념을 **구체적인 화면 형태로 전환**

### 해결하는 사용자 불안
- "시안이란 게 뭔데?"
- "막연한 그림만 주는 거 아냐?"
- "내 가게랑 너무 다른 거 아냐?"

→ 결과의 범위를 미리 보여줌 (참고용, 상상용)

## SECTION 7: Free Until Here

### 역할
- 망설임이 **'진입'으로 바뀌는 지점**
- 선택이 아닌 **경험 진입**을 제안
- 비용 부담 전에 **리스크 제거**

### 카피
> "그래서, 여기까지는 비용이 들지 않습니다."

**사용자 반응 목표**: "안 해볼 이유는 없네."

## SECTION 8: Portfolio (NEW)

### 역할
- 실제 작업물을 통해 **신뢰 형성**
- "이런 결과물이 나오는구나" 직관적 이해
- /portfolio 페이지로의 진입점

### 연출: Marquee 카드 갤러리
**Magic UI Marquee 기반** 좌우로 무한 스크롤되는 카드:
- 호버 시 일시정지
- 카드 클릭 시 상세 페이지 또는 확대
- 2줄 구성 (위: 정방향, 아래: 역방향)

### 카드 구성
```
이미지 (스톡 이미지 필수)
프로젝트명
카테고리 (웹사이트, 랜딩페이지 등)
```

### 섹션 타이틀
> "우리가 만든 것들"

### 하단 CTA
> "더 많은 작업물 보러가기 →"
→ /portfolio 페이지로 이동

### 이미지 가이드
- **스톡 이미지 필수** (플레이스홀더 사용 금지)
- 웹사이트/랜딩페이지 스크린샷 스타일
- 일관된 비율 유지 (16:9 또는 4:3)

## SECTION 9: Why We Work Like This (기존 8번)

### 역할
- "왜 여기까지 무료인가"에 대한 **감정적 이유**
- "왜 이렇게 설계했는가"에 대한 태도 설명
- 지금까지 본 모든 구조를 **하나의 의도로 묶는 구간**

### 연출
- **스토리처럼 슥슥 바뀌면서** 진행
- 큰 여백, 큰 타이포, 에디토리얼 느낌

### 카피 (그대로 사용)

> 웹사이트는 필요하다고들 말합니다.
> 하지만 비용과 복잡함 앞에서
> 시작조차 못 하는 경우를 우리는 너무 많이 봤습니다.

> 그건 웹사이트가 어려워서가 아니라,
> 어렵게 만들어온 관행이
> 당연해졌기 때문이라고 생각했습니다.

> 그래서 우리는
> 더 복잡한 방식을 따르기보다,
> 그동안 당연하게 여겨지던 과정을 하나씩 걷어내기로 했습니다.

> 설명은 어렵지 않게,
> 결정은 쉽게,
> 부담 없이 한 번 시작해볼 수 있도록.

> 보이지 않는 곳에서 구조를 바꾸는 일.
> 고객의 첫 단계를 가볍게 만드는 선택.

> **우리는 Invisible Works입니다.**
